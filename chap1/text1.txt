확률 이론 기초

표본 공간: 샘플 x가 가질 수 있는 모든 값의 집합
확률 밀도 함수: point x를 0과 1사이의 숫자로 매핑하는 함수 p(x)
모수 모델링: 안정적인 P_model(x)를 찾는 데 사용할 수 있는 기법, 유한 개수의 파라미터 Θ를 사용해 기술할 수 있는 밀도 함수 p_Θ(x)의 한 종류
가능도: L(Θ|x) 관측된 포인트 x가 주어졌을 때 Θ의 타당성을 측정하는 함수/L(Θ|x) = p_Θ(x)
        즉, 관측 포인트 x가 주어졌을 때 Θ의 가능도는 포인트 x에서 Θ를 파라미터로 가진 밀도 함수의 값으로 정의
        독립된 샘플로 구성된 데이터셋 전체 X가 있으면 L(Θ|x) = (x∈X)ΠP_Θ(x)
        소수의 곱은 비용이 쎄서 로그로 표시하는 경우가 잦다고한다
        l(Θ|X) = (x∈X)ΣP_Θ(x)
        실제 데이터 생성 분포가 Θ를 매개변수로 가진 모델이라면, Θ 집합의 가능도를 데이터가 발견될 확률이라 정의

최대 가능도 추정: Θ_hat을 추정할 수 있는 기법, Θ_hat = argmax(x, l (Θ|X))
Θ_hat: 관측된 데이터 X를 가장 잘 설명하는 밀도 함수 p_Θ(x)의 파라미터 집합 Θ
일반적으로 신경망은 손실 함수를 최소화하는데,
음의 로그 가능도를 최소화하는 파라미터 집합을 찾는 것과 같다고 말할 수 있다???????
Θ_hat = argmin(Θ, -l (Θ|X)) = argmin(Θ, -logP_Θ(X))

생성 모델링은 최대 가능도 추정의 형태로 생각 가능
Θ는 모델에 담긴 신경망의 가중치
주어진 데이터를 관측할 가능도를 최대화하는 파라미터 값을 찾는것 (혹은 음의 로그 가능도를 최소화하는)
??? 가능도 얘기는 다시 한 번 읽어보자

생성 모델별로 밀도함수 P_Θ(x)를 모델링하는 방식이 다르다
1. 명시적 밀도 함수를 모델링 하고, 밀도함수를 다루기쉽(= 계산가능하도록) 모델을 제약
2. 다루기 쉬운 밀도 함수의 근사치를 명시적으로 모델링
3. 데이터를 직접 생성하는 확률정 과정을 통해 밀도 함수를 암묵적으로 모델링

암묵적 밀도와 명시적 밀도로 나눠서
암묵적은 GAN
명시적은 또다시 둘로 나눠서
다루기 쉬운은 자기회귀나 노멀라이징 플로 모델
나머지는 근사밀도로서 VAE, 에너지 기반, 디퓨전 모델
